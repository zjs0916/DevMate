# Internal Project Guidelines

When building small web projects, prefer simple and readable file structures.

For FastAPI projects:
- Use `src/` layout.
- Keep route handlers small.
- Put business logic in separate modules.
- Use logging instead of print.
- Include a README with setup and run instructions.

For frontend demo projects:
- Use semantic HTML.
- Keep CSS in a separate file.
- Avoid unnecessary frameworks unless the user asks for them.
